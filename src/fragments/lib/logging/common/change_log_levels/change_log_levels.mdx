## Change Default Log Level

Below is an example of setting the default log level to `WARN`:

import iosDefaultLogLevel from '/src/fragments/lib/logging/ios/change_log_levels/10_default_log_level.mdx';
import androidDefaultLogLevel from '/src/fragments/lib/logging/android/change_log_levels/10_default_log_level.mdx';
import commonDefaultLogLevel from '/src/fragments/lib/logging/common/change_log_levels/10_default_log_level.mdx';

<BlockSwitcher>
    <Block name="With Configuration File">
        <Fragments fragments={{all: commonDefaultLogLevel}}/>
    </Block>
    <Block name="With Code">
         <Fragments
            fragments={{
                ios: iosDefaultLogLevel,
                android: androidDefaultLogLevel
            }}/>
    </Block>
</BlockSwitcher>

## Configure Log Level by Category
Each Amplify category can be configure with its own log level.  

Below is an example of setting different log levels for Storage and Auth categories:

import iosCategoryLogLevel from '/src/fragments/lib/logging/ios/change_log_levels/30_category_log_level.mdx';
import androidCategoryLogLevel from '/src/fragments/lib/logging/android/change_log_levels/30_category_log_level.mdx';
import commonCategoryLogLevel from '/src/fragments/lib/logging/common/change_log_levels/30_category_log_level.mdx';

<BlockSwitcher>
    <Block name="With Configuration File">
        <Fragments fragments={{all: commonCategoryLogLevel}}/>
    </Block>      
    <Block name="With Code">
        <Fragments
            fragments={{
                ios: iosCategoryLogLevel,
                android: androidCategoryLogLevel
            }}
        />  
    </Block>
</BlockSwitcher>
