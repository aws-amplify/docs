## Change Default Log Level

Below is an example of setting the default log level to `WARN`:

import iosDefaultLogLevel from '/src/fragments/lib/logging/ios/change_log_levels/10_default_log_level.mdx';
import androidDefaultLogLevel from '/src/fragments/lib/logging/android/change_log_levels/10_default_log_level.mdx';
import commonDefaultLogLevel from '/src/fragments/lib/logging/common/change_log_levels/10_default_log_level.mdx';

<BlockSwitcher>
    <Block name="With Configuration File">
        <Fragments fragments={{all: commonDefaultLogLevel}}/>
    </Block>
    <Block name="With Code">
         <Fragments
            fragments={{
                ios: iosDefaultLogLevel,
                android: androidDefaultLogLevel
            }}/>
    </Block>
</BlockSwitcher>

## Configure Log Level Log by User
Each Amplify authenticated user can be configure with their own unique logging configuration.  The Amplify user's AWS Cognito User Pool Sub is used as the user identifier. 

### Getting the Cognito User Pool Sub
You can use the Amplify Auth category to get the Cognito User Pool Sub.

import iosCurrentUser from '/src/fragments/lib/logging/ios/change_log_levels/50_auth_current_user.mdx';
import androidCurrentUser from '/src/fragments/lib/logging/android/change_log_levels/50_auth_current_user.mdx';

<Fragments
    fragments={{
        ios: iosCurrentUser,
        android: androidCurrentUser
    }}/>
