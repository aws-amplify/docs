## Change Default Log Level
To change the default log level, set the desired log level `defaultLogLevel` field in the configuration file.

Below is an example of setting the default log level to `WARN`:
```json
{
    "awsCloudWatchLoggingPlugin": {
        "enable": true,
        "logGroupName": "<log-group-name>",
        "region": "us-east-1",
        "localStoreMaxSizeInMB": 1,
        "flushIntervalInSeconds": 60,
        "loggingConstraints": {
            "defaultLogLevel": "WARN"
        }
    }
}
```

## Change local storage maximum size
To change the maximum amount of local logs are stored before sending logs to AWSCloudWatch, change the field `localStoreMaxSizeInMB` in the configuration file.

Below is an example of setting the local storage size to 2 MB:
```json
{
    "awsCloudWatchLoggingPlugin": {
        "enable": true,
        "logGroupName": "<log-group-name>",
        "region": "us-east-1",
        "localStoreMaxSizeInMB": 2,
        "flushIntervalInSeconds": 60,
        "loggingConstraints": {
            "defaultLogLevel": "WARN"
        }
    }
}
```

## Change automatic log flush interval
To change the time interval when logs are automatically flushed and sent to AWSCloudWatch, change the field `flushIntervalInSeconds` in the configuration file.

Below is an example of setting the time interval to 120 seconds:
```json
{
    "awsCloudWatchLoggingPlugin": {
        "enable": true,
        "logGroupName": "<log-group-name>",
        "region": "us-east-1",
        "localStoreMaxSizeInMB": 1 MB,
        "flushIntervalInSeconds": 120,
        "loggingConstraints": {
            "defaultLogLevel": "WARN"
        }
    }
}
```

## Configure Log Level by Category
Each Amplify category can be configure with its own log level.  

Below is an example of setting different log levels for Storage and Auth categories:
```json
{
    "awsCloudWatchLoggingPlugin": {
        "enable": true,
        "logGroupName": "<log-group-name>",
        "region": "us-east-1",
        "localStoreMaxSizeInMB": 1 MB,
        "flushIntervalInSeconds": 120,
        "loggingConstraints": {
            "defaultLogLevel": "WARN",
            "categoryLogLevel": {
                    "AUTH": "VERBOSE",
                    "STORAGE": "DEBUG"
            }
        }
    }
}
```

## Configure Log Level Log by User
Each Amplify authenticated user can be configure with their own unique logging configuration.  The Amplify user's AWS Cognito User Pool Sub is used as the user identifier. 

Below is an example of setting different a default log level and category level for an authenticated user with AWS Cognito User Pool sub `xyz-123`
```json
{
    "awsCloudWatchLoggingPlugin": {
        "enable": true,
        "logGroupName": "<log-group-name>",
        "region": "us-east-1",
        "localStoreMaxSizeInMB": 1 MB,
        "flushIntervalInSeconds": 120,
        "loggingConstraints": {
            "defaultLogLevel": "WARN",
            "categoryLogLevel": {
                    "AUTH": "WARN",
                    "STORAGE": "WARN"
            },
            "userLogLevel": {
                "xyz-123": {
                    "defaultLogLevel": "DEBUG",
                    "categoryLogLevel": {
                        "AUTH": "VERBOSE",
                        "DATASTORE": "VERBOSE",
                        "API": "VERBOSE"
                    }
                }
            }
        }
    }
}
```
