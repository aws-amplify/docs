Get the properties of the file using `getProperties` method.

```typescript
await Storage.getProperties(key: 'srcKey');
```

The format of the response looks like this:

```js
 {
    contentType: "image/jpeg",
    contentLength: 6873,
    eTag: "\"56b32cf4779ff6ca3ba3f2d455fa56a7\"",
    lastModified: Wed Apr 19 2023 14:20:55 GMT-0700 (Pacific Daylight Time) {},
    metadata: { owner : 'aws'}
}

```

To utilize Encryption, the following options can be passed in to `Storage.getProperties`:

```typescript
const SSECustomerAlgorithm = 'string';
const SSECustomerKey = new Buffer('...') || 'string';
const SSECustomerKeyMD5 = 'string';

const result = await Storage.getProperties(
  { key: 'srcKey' },
  {
    SSECustomerAlgorithm,
    SSECustomerKey,
    SSECustomerKeyMD5
  }
);
```

### File Access Levels

You can choose to configure [file access `level`](https://docs.amplify.aws/lib/storage/configureaccess/q/platform/js) as follows:

```typescript
const level = 'private' || 'protected' || 'public';
const result = await Storage.getProperties('welcome.png', { level }); // defaults to `public`
```

- `public`: Accessible by all users of your app. Files are stored under the `public/` path in your S3 bucket.
- `protected`: Readable by all users, but writable only by the creating user. Files are stored under `protected/{user_identity_id}/` where the `user_identity_id` corresponds to the unique Amazon Cognito Identity ID for that user.
- `private`: **Only accessible for the signed in user**. Files are stored under `private/{user_identity_id}/` where the `user_identity_id` corresponds to the unique Amazon Cognito Identity ID for that user.
